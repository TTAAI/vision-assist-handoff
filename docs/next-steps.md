# 下一步计划

## 阶段 1：跑通手机摄像头闭环

目标：

```text
红米 Note 14 手机摄像头 -> /analyze -> YOLO识别 -> TTS播报
```

任务：

1. 做 `phone-camera-only` APK。
2. App 中服务器地址可配置，不再写死。
3. 服务端兼容原始 JPEG 上传。
4. 服务端返回 `message` 和 `speak` 双字段。
5. 真机点击“上传测试”，确认 Featurize 日志出现 `POST /analyze 200 OK`。

验收：

- 手机打开 App 不闪退。
- 摄像头预览可见。
- 点击上传后服务器收到请求。
- 手机播报识别结果。

## 阶段 2：连续识别

目标：

- 手机预览 30fps。
- 上传 3-5fps 起步，稳定后提高到 8-10fps。
- 语音播报节流，避免重复播报。

建议：

- YOLO 检测：8-10fps。
- 深度估计：2-4fps。
- OCR：按需触发。
- 同类播报 2-4 秒内只播一次。

## 阶段 3：算法增强

优先模块：

- YOLO：人、车、自行车、摩托车、交通灯、椅子、常见障碍物。
- Depth Anything V2：近/中/远相对距离。
- PaddleOCR：中文路牌、招牌、站牌，按需触发。

后续数据：

- 采集台阶、路沿、水坑、施工障碍、斑马线、夜间、逆光、人流密集场景。
- 用 GroundingDINO/SAM 辅助标注。
- 微调自有 YOLO/YOLO-seg 模型。

## 阶段 4：外置摄像头

在手机摄像头闭环稳定后再恢复 UVC：

- 单独调试 UVC 库。
- 用标准 Gradle/Android Studio 构建，避免旧 `dx` 工具链。
- 外接摄像头到货后只替换采集源，不改服务器协议。
